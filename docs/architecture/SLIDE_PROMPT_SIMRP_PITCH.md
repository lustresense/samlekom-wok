# Prompt AI Slide - SIMREKAP Pitch Deck (Versi Panjang)

Gunakan prompt ini ke Canva AI / Gamma / Tome.

---

'Buat presentasi berbahasa Indonesia untuk kebutuhan pitching proyek akhir + pengajuan operasional ke instansi.

Nama solusi:
SIMREKAP - Sistem Informasi Manajemen RElawan KAmpung Pancasila

Tagline:
The Pillar-Balance & Maturity Engine

Tujuan presentasi:
1. Menjelaskan konteks Program Kampung Pancasila Surabaya.
2. Menjelaskan masalah lapangan yang belum tertata datanya.
3. Menawarkan solusi SIMREKAP berbasis gamifikasi kampung-centric.
4. Menjelaskan struktur aktor secara tegas dan tidak tertukar.
5. Menekankan kontribusi relawan: sertifikat per kegiatan + reward poin.
6. Menunjukkan manfaat operasional dan arah implementasi bertahap.

## Aturan istilah (WAJIB - jangan dilanggar)
- Jangan pakai istilah "warga" sebagai aktor umum.
- Gunakan istilah ini secara konsisten:
  - Relawan = warga yang tinggal di Surabaya (termasuk mahasiswa/perantau yang tinggal/beraktivitas di Surabaya).
  - KSH = entitas operasional lapangan (kepanitiaan), terpisah dari relawan umum.
- KSH tidak membuat draft kegiatan.
- KSH tidak approve kegiatan.
- KSH fokus mengelola kegiatan lapangan, menandai kegiatan selesai, dan mengisi output/laporan operasional.
- Moderator ada 3 tier:
  - Tier 1: ASN Pendamping (inisiasi draft + monitoring kampung binaan).
  - Tier 2: Verifikator wilayah dengan badge:
    - Badge A = Kelurahan (scope 1 kelurahan).
    - Badge B = Kecamatan (scope 1 kecamatan, mencakup banyak kelurahan).
  - Tier 3: OPD (monitoring agregat dan insight kebijakan).
- Admin adalah back-office sistem, bukan pengampu operasional kampung.
- Mitra eksternal tidak login. Mitra submit request publik dan diproses moderator sesuai scope kontribusi.

## Batasan konten
- Fokus leaderboard kampung, bukan leaderboard individu.
- Tegaskan 1 kegiatan = 1 sertifikat relawan.
- Tegaskan poin relawan bisa ditukar voucher manfaat layanan Surabaya (poin berkurang saat ditukar).
- Sebutkan fitur early warning ketimpangan pilar sebagai penguatan lanjutan (dalam kajian), bukan fitur fully live.
- Jika ada fitur yang masih proses kajian, beri label: "Status: Dalam Kajian".

## Gaya visual
- Formal simple profesional
- Warna bebas asal mudah dipahami
- Nuansa startup pitch: clean, tegas, data-first, problem-solution-impact.
- Gunakan diagram alur, role map, timeline, dan kartu ringkasan.
- Hindari tampilan terlalu ramai.

## Jumlah slide
Buat 22 slide (versi lengkap, bukan ringkas).

## Format output per slide (WAJIB)
- Headline:
- Visual:
- Poin Utama (3-5 bullet):
- Penjelasan Singkat (2-4 kalimat):

## Urutan 22 slide
1. Judul Presentasi
   - Nama solusi, owner, tim, konteks pengajuan.

2. Kenapa Topik Ini Penting
   - Urgensi tata kelola kampung berbasis data.

3. Kampung Pancasila Surabaya Itu Apa
   - Definisi program tingkat kampung/RW.

4. Empat Pilar Program
   - Lingkungan, Ekonomi, Sosial Budaya, Kemasyarakatan.

5. Kondisi Lapangan Saat Ini
   - Aktivitas tinggi, data belum tertata.

6. Gap Utama yang Harus Ditutup
   - Verifikasi, akuntabilitas, dan ketimpangan pilar.

7. Visi Solusi
   - SIMREKAP sebagai mesin operasional kampung.

8. Solusi Inti: Gamifikasi Kampung-Centric
   - Fokus penilaian di kampung, bukan individu.

9. Pillar-Balance Engine
   - Bagaimana sistem mendorong pilar yang tertinggal.

10. XP Kampung dan Leaderboard Kampung
   - Fungsi sebagai indikator kinerja wilayah.

11. Struktur Aktor Besar
   - Relawan, KSH, Moderator, Admin, Mitra.

12. Aktor 1: Relawan
   - Siapa relawan, kontribusi utama, tujuan partisipasi.

13. Aktor 2: KSH
   - KSH sebagai panitia operasional kegiatan, bukan pembuat draft.

14. Aktor 3: Moderator Tier 1
   - ASN pendamping: inisiasi draft dan monitoring kampung binaan.

15. Aktor 4: Moderator Tier 2 + Badge
   - Badge A Kelurahan vs Badge B Kecamatan dan beda kewenangan.

16. Aktor 5: Moderator Tier 3 dan Admin
   - OPD untuk agregasi kebijakan, admin untuk back-office sistem.

17. Fitur Utama Relawan (WAJIB disorot)
   - Join kegiatan, lapor kontribusi, lihat poin, lihat riwayat kontribusi.

18. Sertifikat Relawan
   - 1 kegiatan 1 sertifikat; alur penerbitan sertifikat; status otomasi (jika masih dikaji, beri label).

19. Reward Poin Relawan
   - Poin -> voucher manfaat Surabaya (contoh: feeder Wira-wiri, Suroboyo Bus), poin berkurang saat redeem.

20. Alur End-to-End Kegiatan
   - Draft ASN T1 -> Approval T2 -> Partisipasi -> Operasional KSH -> Laporan -> Verifikasi.

21. Alur Kolaborasi Mitra
   - Submit publik non-login -> routing scope kelurahan/kecamatan/kota -> review moderator.

22. Penutup dan Ajakan Implementasi
   - Nilai manfaat, status implementasi, roadmap penguatan fitur, call to action.'
